# Docs

## Allgemein

| Attribut            | Wert                                                                   |
| ------------------- | ---------------------------------------------------------------------- |
| Systematischer Name | `docs`                                                                 |
| Funktionalität      | Dokumentation der Schülereigenen Softwareprojekte                      |
| Stack               | Docusaurus                                                             |
| Deployment          | GitHub Pages                                                           |
| URL                 | [https://docs.birklehof.space/](https://docs.birklehof.space/)         |
| Code                | [https://github.com/Birklehof/docs](https://github.com/Birklehof/docs) |

## Wichtige Befehle

- `yarn start` - Startet den Entwicklungsserver
- `yarn build` - Erstellt die statischen Dateien für die Website
- `yarn serve` - Startet einen lokalen Server, um die statische Seite zu testen
- `yarn deploy` - Veröffentlicht die Website auf GitHub Pages

Nutze `--host 0.0.0.0` um den Entwicklungsserver bzw den lokalen Server für andere Geräte im Netzwerk verfügbar zu machen (z.B. wenn du auf einer
virtuellen Maschine entwickelst).

## Deployment

Um Änderungen an der Dokumentation zu veröffentlichen, kannst du einfach den Befehl `yarn deploy` ausführen. Beachte, dass du hierfür am besten über
[SSH mit GitHub verbunden](https://docs.github.com/en/authentication/connecting-to-github-with-ssh) sein solltest.
